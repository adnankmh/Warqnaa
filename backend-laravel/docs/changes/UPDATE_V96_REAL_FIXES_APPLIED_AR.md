# تقرير تحديث v96 — تعديلات ظاهرة ومباشرة

تم تطبيق التعديلات داخل الملفات المؤثرة فعليًا، مع كسر كاش المتصفح عبر تغيير روابط CSS/JS إلى `?v=96-real-fixes` حتى تظهر التعديلات بعد التشغيل.

## الملفات التي تغيّرت فعليًا
- `resources/views/layouts/app.blade.php`
- `resources/views/room/index.blade.php`
- `resources/views/room/show.blade.php`
- `resources/views/store/index.blade.php`
- `resources/views/admin/index.blade.php`
- `app/Http/Controllers/RoomController.php`
- `public/assets/css/app.css`
- `public/assets/js/app.js`
- `server/index.js`

## أهم الإصلاحات
- تثبيت سرعة الدور على 5 / 7 / 10 ثوانٍ فقط ومنع ظهور أرقام قديمة مثل 998 ثانية.
- إظهار غرف اللعبة فقط إذا كان فيها لاعب حقيقي متصل، وإغلاق/إخفاء الغرفة عند خروج جميع اللاعبين الحقيقيين.
- زر الخروج يرجع مباشرة إلى صفحة غرف نفس اللعبة.
- رجوع زر إنشاء اللعبة إلى يمين صفحة الغرف مع نافذة إنشاء من منتصف الصفحة أيضًا.
- حذف واجهة الثيمات واللغات من الواجهة الرئيسية بالكامل.
- منع الحاجة لتأكيد بداية اللعبة، والبداية تتم مباشرة من الزر.
- بعد انتهاء الجولة يتم احتساب النقاط والانتقال تلقائيًا إلى الجولة التالية بدون زر توزيع/بدء جديد.
- تجهيز الصوت بين اللاعبين عبر WebRTC Signaling في `server/index.js` عند تشغيل `npm run socket`، مع خصم 100 توكنز عند إنشاء/دخول غرفة صوتية.
- إصلاح معاينات المتجر: الطاولات وحدها، ظهر الورق وحده، وألوان الدردشة وحدها بدون تداخل.
- جعل كل لون دردشة يعرض “كلمة تجريبية” بنفس اللون، ويستخدم اللون في دردشة اللعبة والأصدقاء بعد التفعيل.
- تحسين تبويبات إدارة المتجر وتقليل النزول للأسفل، مع معاينات أوضح لكل نوع.
- دعم رفع صورة الطاولة وظهر الورق من الإدارة وظهورها في اللعبة.
- تحسين Responsive للموبايل والتابلت.
- إضافة أصوات/تفاعلات UI أكثر في الأزرار والتنقلات.

## التشغيل
- Laravel ما زال على البورت `8000`.
- لتفعيل الصوت بين الأجهزة شغّل أيضًا:

```bash
npm install
npm run socket
```

ثم شغّل Laravel من `start-windows.bat` أو:

```bash
php artisan serve --host=127.0.0.1 --port=8000
```

## الفحص
- تم فحص ملفات PHP عبر `php -l` بدون أخطاء.
- تم فحص `public/assets/js/app.js` و `server/index.js` عبر `node --check` بدون أخطاء.
