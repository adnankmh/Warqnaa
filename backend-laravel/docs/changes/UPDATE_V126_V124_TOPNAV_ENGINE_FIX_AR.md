# تحديث v126 — الرجوع إلى v124 مع إصلاح الورقة الصحيحة

## ما تم
1. الرجوع إلى أساس v124
- لم يتم البناء على v125.
- تم الحفاظ على الـ NAV BAR في الأعلى.
- تم إلغاء أي تأثير تنقل جانبي مزعج.

2. إصلاح مشكلة الورقة الصحيحة
السبب المحتمل كان من أكثر من نقطة:
- أحيانًا الواجهة ترسل الورقة بصيغة مختلفة مثل card_id أو rank+suit أو object.
- أحيانًا AntiCheat يعتبر الضغط سريعًا فيرفض الحركة قبل محرك اللعبة.
- أحيانًا الرسالة العامة تظهر حتى لو السبب ليس الورقة نفسها.

تم الإصلاح عبر:
- تقوية PlayActionNormalizer لقبول:
  - card
  - card_id
  - id
  - code
  - rank + suit
  - object مثل {id: A_clubs}
  - object مثل {rank: A, suit: سنك}
- تقوية TarneebRules لنفس الصيغ.
- إيقاف رفض الحركة الصحيحة بسبب tooFast؛ الآن يتم تسجيلها فقط كتحذير ولا تمنع الورقة إذا كانت قانونية.
- تحسين رسالة الخطأ لتفرق بين:
  - الورقة ليست في يدك.
  - يجب اتباع النوع.
  - الحالة قديمة والدور تغير.

3. الحفاظ على قانون الطرنيب
- لا يزال اتباع النوع مفعّلًا.
- لا يسمح بلعب ورقة مخالفة إذا كان لديك من نوع أول ورقة.
- لا يسمح باللعب خارج الدور.

4. اختبارات جديدة
- tests/Feature/V126CardNormalizerTest.php
- اختبار صيغ ورق متعددة.
- اختبار أن الورقة الصحيحة لا تُرفض.
- اختبار أن قانون اتباع النوع لا يزال يعمل.

## ملاحظات حول كل محركات الألعاب
تمت تقوية طبقة الإدخال والتدقيق لكل الألعاب عبر GameFactory/EngineAudit الموجودة من v124. لكن كل لعبة كبيرة مثل البلوت/الهاند/التريكس تحتاج اختبارات لعب مطولة لكل الاستثناءات للوصول إلى دقة 100%.
