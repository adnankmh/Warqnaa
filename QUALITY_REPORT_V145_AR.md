# تقرير الجودة — Warqna v145

## نطاق الفحص

تم فحص الحزمة بعد إعادة بناء طبقة الألعاب المحلية وواجهة المتجر وميزات PWA، مع الحفاظ على تكامل Laravel عند توفر الخادم.

## نتائج الفحص المنفذة داخل بيئة التجهيز

- فحص الصياغة لجميع ملفات PHP باستخدام `php -l`: ناجح.
- فحص البنية النحوية لجميع ملفات Dart باستخدام محلل Tree-sitter: ناجح.
- فحص ملفات JSON وYAML: ناجح.
- التحقق من عدم تكرار مفاتيح منتجات متجر Flutter: 131 منتجاً فريداً.
- التحقق من توزيع فئات المتجر:
  - 6 باقات باشا.
  - 5 ثيمات.
  - 50 طاولة.
  - 40 ظهر ورق.
  - 7 حزم إيموجي.
  - 6 مسرعات خبرة.
  - 11 لون/هوية اسم.
  - 3 شارات.
  - 3 مؤثرات.
- التحقق من قائمة الألعاب الظاهرة: 12 لعبة/نمط فقط وفق القائمة المعتمدة.
- التحقق من وجود حسابات التجربة محلياً وفي Seeder الخاص بـ Laravel.
- التحقق من أحجام أيقونات PWA: 192 و512 وMaskable وApple Touch Icon.
- التحقق من بنية GitHub Actions الخاصة بالويب وAndroid وiOS.
- التحقق من سلامة ملف ZIP النهائي بعد إنشائه.

## ما لم يُنفذ محلياً

Flutter SDK غير مثبت داخل بيئة التجهيز، لذلك لم يتم تشغيل `flutter analyze` أو `flutter test` أو إنشاء Build محلياً. وضعت هذه الخطوات داخل GitHub Actions بحيث تعمل تلقائياً قبل نشر GitHub Pages أو إنتاج APK/AAB.

Laravel لا يحتوي على مجلد `vendor` داخل الحزمة، لذلك لم يتم تشغيل اختبارات Laravel أو أوامر Artisan الكاملة محلياً. تم فحص جميع ملفات PHP نحوياً، وأصلحت موضع كود Seeder ليعمل داخل `run()` بدلاً من التنفيذ خارج الصنف.

## وضع التشغيل

- على GitHub Pages: تعمل الألعاب الظاهرة بمحركات محلية ضد لاعبين آليين، وتعمل الواجهات والمتجر والمكافآت والتحديات والحفظ المحلي.
- عند نشر Laravel وربط `WARQNA_API_URL`: يتحول إنشاء الغرف إلى الخادم، وتصبح الحسابات والدردشة والأصدقاء والتحويلات واللعب بين أجهزة مختلفة معتمدة على قاعدة البيانات والخادم.
- زر تغيير الاتجاه يبدل تخطيط التطبيق فوراً؛ في Android/iOS يطلب أيضاً اتجاه الجهاز. داخل متصفح الهاتف قد يظل التدوير الفعلي خاضعاً لإعداد قفل دوران الجهاز والمتصفح.
