# ورقنا v145 — ابدأ من هنا

هذه النسخة مبنية على v144 وتضيف تشغيل جميع الألعاب الظاهرة داخل Flutter Web/PWA حتى عند عدم توفر Laravel.

## ما الذي تغيّر؟

- أزيلت من الواجهة: الدومينو، جاكارو، الشطرنج، سوليتير التنافسي، البناكل كلاسيك، وطاولة الزهر.
- أصبحت القائمة النهائية 12 لعبة/نمطاً: طرنيب، تركس، هاند، بناكل، بلوت، باصرة، طرنيب 400، طرنيب سوري، تركس كمبلكس، هاند سعودي، هاند شراكة، وتركس شراكة.
- عند توفر Laravel تستخدم اللعبة المحرك الخادمي. عند غيابه تنتقل تلقائياً إلى المحرك المحلي، ولا تظهر صفحة «محرك Laravel جاهز».
- المتجر يحتوي 131 عنصراً، منها 50 طاولة و40 ظهر ورق، مع معاينة وشراء وتفعيل.
- إضافة تثبيت PWA على Android وiPhone، وأيقونات جديدة، ودعم الوضع الطولي/العرضي.

## تجربة سريعة عبر GitHub Pages

1. انسخ محتويات هذه الحزمة فوق مستودع Warqna في GitHub Desktop.
2. نفّذ Commit ثم Push إلى فرع `main`.
3. افتح GitHub ثم `Actions`.
4. شغّل `Build and deploy Flutter Web`.
5. بعد ظهور العلامة الخضراء افتح:

```text
https://adnankmh.github.io/Warqna/
```

لا يوجد Workflow للمعاينة القديمة في هذه النسخة، حتى لا يستبدل تطبيق Flutter بالخطأ.

## الدخول المحلي

اضغط «اختيار حساب تجريبي آخر» من شاشة الدخول، ثم اختر الحساب واضغط «تشغيل محلي».

الحساب الإداري:

```text
Adnan
Adnan123
```

رصيد المدير: `1000000000000000000` توكن.

## تشغيل Laravel محلياً

داخل `backend-laravel`:

```bat
setup-windows.bat
start-windows.bat
```

الرابط الافتراضي:

```text
http://127.0.0.1:8006
```

ولتوصيل GitHub Pages بخادم حقيقي، أنشئ Repository Variable باسم:

```text
WARQNA_API_URL
```

وقيمته مثالاً:

```text
https://api.example.com/api/mobile/v1
```

## تثبيت التطبيق

### Android

افتح رابط GitHub Pages في Chrome. سيظهر شريط «ثبّت تطبيق ورقنا». اضغط «تثبيت». ويمكن أيضاً استخدام قائمة Chrome ثم `Install app`.

### iPhone

افتح الرابط في Safari. ستظهر تعليمات التثبيت. اضغط زر المشاركة ثم `Add to Home Screen`.

## بناء Android وiOS

- Android: شغّل Workflow باسم `Build Android APK and AAB`.
- iOS غير موقع: شغّل `Build iOS unsigned`.
- النشر النهائي على App Store يحتاج حساب Apple Developer وتوقيعاً عبر Xcode/macOS.

## بعد استبدال النسخة في GitHub

1. احذف الملفات القديمة من مجلد المستودع المحلي، ثم انسخ محتويات هذه الحزمة إلى جذر المستودع.
2. نفّذ `Commit to main` ثم `Push origin` من GitHub Desktop.
3. شغّل Workflow الجديد بعد الرفع؛ لا تعِد تشغيل عملية قديمة حمراء.
4. على الهاتف، امسح Cache الموقع أو افتحه مرة في نافذة خاصة إذا بقيت نسخة قديمة من Service Worker.
5. جرّب لعبة غير الطرنيب بدون Laravel: يجب أن يظهر في العنوان `محلي ذكي` بدلاً من صفحة خطأ Laravel.
