# رفع وبناء Warqnaa V0.3.1 / v182 عبر GitHub

1. ارفع محتويات المشروع كاملة، ومن ضمنها `.github/workflows`.
2. شغّل **Production Release Check** ثم **Backend CI**.
3. شغّل مسار Flutter Web أو Android أو iOS المطلوب.
4. اضبط `WARQNA_API_URL` على رابط Laravel الفعلي؛ GitHub Pages وحده لا يوفر اقتصادًا مركزيًا أو غرفًا حقيقية بين الأجهزة.
5. نفّذ migrations الخاصة بـ v182 قبل تفعيل دولاب الحظ على الإنتاج.
6. استخدم HTTPS، واضبط CORS والجلسات والكاش وقاعدة البيانات وطوابير المهام.
7. استبدل معرفات AdMob الاختبارية بمعرفات الإنتاج قبل النشر في المتاجر.
8. لا ترفع `.env` أو مفاتيح التوقيع أو أسرار Firebase إلى المستودع.

الإصدار المتوقع في جميع المسارات: `0.3.1+182`.
