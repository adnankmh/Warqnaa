# خطوات الإنتاج التالية — Warqna v152

## أنجز داخل الكود

- اختيار لعبة عادية أو صوتية لجميع الألعاب.
- إنشاء غرف عامة، للأصدقاء، أو خاصة بكلمة سر.
- سرعات دور 5 و7 و10 ثوانٍ.
- قائمة غرف مفتوحة ودخول برمز الغرفة.
- منع الحساب من دخول غرفة ثانية قبل مغادرة الأولى.
- WebRTC للصوت، مع REST signaling موثّق عبر Laravel.
- صلاحيات الميكروفون للأندرويد وiOS.
- تنظيف تلقائي للحضور والإشارات الصوتية القديمة.
- وضع محلي آمن للعرض على GitHub Pages.

## يحتاج منك حسابات أو خدمات خارجية

1. نطاق مثل `warqna.com`.
2. خادم Linux أو Laravel Cloud لتشغيل Laravel وMySQL/Redis.
3. خادم TURN مُدار أو Coturn على خادم مستقل.
4. مفاتيح AdMob الحقيقية عند الانتقال من الاختبار للإنتاج.
5. حساب Google Play Developer وحساب Apple Developer للنشر في المتاجر.
6. مفاتيح OAuth الخاصة بـGoogle وApple وFacebook عند تفعيل الدخول الاجتماعي الحقيقي.

## خطة نشر عملية

1. نشر Backend على `staging-api.warqna.com`.
2. إضافة متغير `WARQNA_API_URL` في GitHub.
3. اختبار غرفة عادية بأربعة أجهزة.
4. اختبار غرفة صوتية بأربعة أجهزة على شبكات مختلفة.
5. اختبار الانقطاع والعودة والكتم والدخول برمز وكلمة السر.
6. إنشاء Closed Beta قبل النشر العام.
7. نقل الخادم إلى `api.warqna.com` بعد نجاح الاختبارات.
