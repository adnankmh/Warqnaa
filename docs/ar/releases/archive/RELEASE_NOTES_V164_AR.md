# ملاحظات إصدار Warqna v164

الإصدار: **1.65.0+165**

## إصلاح تشغيل APK

- نقل `WidgetsFlutterBinding.ensureInitialized()` و`runApp()` إلى المنطقة المحمية نفسها.
- تشغيل التطبيق فوراً دون انتظار SharedPreferences أو الشبكة أو الإعلانات أو WebRTC.
- تأجيل قراءة البيانات المحلية إلى ما بعد أول إطار.
- إضافة استرداد آمن عند وجود بيانات محلية قديمة أو تالفة.
- إزالة تهيئة AdMob التلقائية من مسار بدء التطبيق؛ تتم التهيئة فقط عند طلب إعلان مكافأة.
- تثبيت AdMob App ID تجريبي صحيح داخل APK التجريبي.
- التحقق الصارم من AdMob App ID وRewarded Unit ID قبل استخدامهما في AAB.
- إضافة صلاحيات الشبكة والصوت المطلوبة والتحقق من MainActivity وSDK levels.
- إضافة فحص توقيع ومحاذاة وسلامة APK قبل رفع Artifact.

## الملفات الناتجة

- `warqna-v164-safe.apk`: للتثبيت المباشر والاختبار.
- `app-release.aab`: للنشر عبر Google Play بعد ضبط المتغيرات الإنتاجية الصحيحة.
