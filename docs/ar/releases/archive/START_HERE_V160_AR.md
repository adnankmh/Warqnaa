# ابدأ من هنا — Warqna v160

هذه النسخة تعالج الأخطاء المتبقية في **Production Release Gate** واختبارات Laravel بعد أن نجح 54 اختبارًا وفشل اختباران فقط في v159.

## الإصلاح الجذري للإصدار

كان Workflow يفحص إصدار v159، لكنه يحتوي على السطر القديم:

```text
WARQNA_APP_BUILD=158
```

تم إلغاء هذا الأسلوب بالكامل. أصبح الملف التالي هو المصدر الوحيد للإصدار:

```text
RELEASE_VERSION.json
```

وتقرأ منه Workflows الخاصة بـ Android وWeb وiOS آليًا. كما يفحص:

```text
tools/verify_release_versions.py
```

جميع نسخ Flutter وLaravel وملفات البيئة وGitHub Actions قبل السماح بالإصدار.

## إصلاح تحويل التوكنز

كان الكود يستدعي:

```php
$sender->wallet()->fresh()
```

لكن `wallet()` تعيد علاقة `HasOne` وليست نموذج Wallet، لذلك ظهر HTTP 500. أصبح الكود يجلب نموذج المحفظة فعليًا عبر:

```php
$sender->wallet()->firstOrFail()
```

مع التحقق من الأرصدة النهائية للمرسل والمستلم والمدير.

## إصلاح robots.txt

أصبح اختبار `Content-Type` غير حساس لحالة الأحرف في `UTF-8`، لأن `UTF-8` و`utf-8` متكافئان ولا ينبغي أن يفشل البناء بسبب اختلاف التنسيق فقط.

## قبل الرفع

شغّل:

```text
CHECK_V160_WINDOWS.bat
```

ثم ارفع محتويات الحزمة إلى جذر المستودع مع إبقاء مجلد `.git` الموجود لديك.
