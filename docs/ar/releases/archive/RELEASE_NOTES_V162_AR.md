# ملاحظات إصدار Warqna v162

## دورة إلغاء الحساب

- تغيير النص من «حذف الحساب» إلى «إلغاء الحساب» في التطبيق والصفحات القانونية.
- تأكيد واضح قبل الإلغاء.
- مهلة استعادة ثابتة لا تقل عن 30 يوماً.
- إبطال جميع جلسات API عند الإلغاء.
- إعادة التفعيل تلقائياً عند تسجيل الدخول بكلمة المرور أو مزود اجتماعي.
- منع إلغاء حساب المدير.
- تحويل المسار القديم `DELETE /account` إلى جدولة آمنة بدلاً من الحذف المباشر.
- تعطيل أي حذف عام للحسابات غير النشطة.

## إصلاح GitHub Actions

السجل المرفق كان يحتوي على رسائل `info` من Flutter Analyzer، مثل استخدام APIs مهملة واقتراحات `const`، وليس أخطاء Compile فعلية. تم:

- استبدال `withOpacity` بـ `withValues(alpha: ...)`.
- استبدال `Color.value` بـ `toARGB32()`.
- استخدام `initialValue` في `DropdownButtonFormField`.
- إصلاح الأقواس و`BuildContext` بعد العمليات غير المتزامنة.
- إضافة Wrapper يميز بين الأخطاء/التحذيرات الحقيقية وبين معلومات التنسيق.
