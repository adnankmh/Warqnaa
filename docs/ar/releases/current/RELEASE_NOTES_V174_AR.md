# ملاحظات إصدار Warqna v174

## إصلاحات البناء والتحليل

- تعريف `loginWithSocialProvider` وربطه بواجهات Google وApple وFacebook الموجودة في الخادم.
- إزالة حالة dead code من شاشة الدخول.
- معالجة تحذيرات الدوال المحلية القديمة مع إبقائها لأغراض ترحيل البيانات.
- إضافة واجهة عامة داخل `AppController` لتحديث المستمعين بدلاً من استدعاء `notifyListeners` المحمي من Extension.
- إصلاح استخدام `BuildContext` بعد العمليات غير المتزامنة.

## الدخول المباشر إلى الألعاب

- إنشاء الغرفة ينقل اللاعب مباشرة إلى واجهة اللعبة المتصلة بالخادم.
- إشعار الدعوة يحمل رمز الغرفة ونوع اللعبة.
- تمت إضافة Preview API آمن لتحديد نوع الغرفة قبل الانضمام، مع دعم الإشعارات القديمة التي تحمل رمز الغرفة فقط.
- تمت إضافة Navigator مركزي لمعالجة إشعارات التطبيق المفتوح والخلفية والتشغيل البارد.

## الاتجاه والشاشة

- أزيل التحويل التلقائي إلى الوضع الأفقي عند فتح غرفة.
- يبقى التطبيق في الوضع الطولي افتراضياً.
- يتحول إلى الوضع الأفقي فقط عندما يختاره اللاعب، ويتم حفظ الاختيار وتطبيقه عند التشغيل التالي.

## الخبرة والتقدم

- تطبيق مضاعفات المستوى الجديدة على Flutter وLaravel بنفس المعادلة.
- الخادم يمنح نقاط كل جولة لجميع اللاعبين الحقيقيين فقط.
- كل لاعب يستقبل نافذة مكافأته الخاصة، ويتم منع كشف مكافآت اللاعبين الآخرين داخل حالة الغرفة.
- تحديث عدادات XP ونقاط الجولة والمنافسة والمجموعة فور وصول نتيجة الجولة.
- إرسال إجماليات التقدم للاعبين الحقيقيين داخل بيانات الغرفة، وإرجاع صفر للبوتات.

## اختبارات Laravel

- تحديث اختبارات الكتالوج القديمة من 90 إلى 140 طاولة لأن الإصدار توسعي.
- جعل اختبار تاريخ الحزمة اليومية متوافقاً مع SQLite وMySQL باستخدام `whereDate`.
- منع تضارب المفتاح الفريد للعبة `basra` باستخدام `updateOrCreate` داخل الاختبار.
