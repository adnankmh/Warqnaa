# تقرير جودة Warqna v168

يركز هذا الإصدار على إصلاح عقد التحقق في CI بعد تنظيم المشروع. كان فحص v167 صحيحًا في الحزمة المضغوطة لكنه فشل داخل GitHub Actions لأن عملية checkout تنشئ `.git` في الجذر.

تم فصل العناصر المسموح بها في المشروع عن بيانات المستودع التشغيلية، وإضافة اختبار ذاتي يثبت أن `.git` وملفات Git القياسية لا تُعد فوضى، بينما يبقى أي ملف عشوائي غير معتمد سببًا للفشل. كما تم تشغيل الفحص محليًا في حالتين: جذر نظيف، وجذر يحاكي GitHub Actions ويحتوي على `.git`.
