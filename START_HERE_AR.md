# ابدأ من هنا — Warqnaa V0.3

## فك الجزأين

الجزءان المضغوطان متكاملان وليسا نسختين منفصلتين. فك **PART-1** و**PART-2** داخل المجلد نفسه مع الحفاظ على بنية المجلدات. لا تشغّل المشروع من جزء واحد فقط.

## أسرع مسار موصى به

1. ارفع محتوى الجزأين بعد دمجه إلى مستودع GitHub واحد.
2. من تبويب **Actions** شغّل `Production Release Check`.
3. شغّل `Backend CI` عند استخدام خادم Laravel.
4. شغّل `Build and deploy Flutter Web` لنشر نسخة GitHub Pages.
5. شغّل `Build Android APK and AAB` لإنتاج ملفات Android.
6. اضبط متغير `WARQNA_API_URL` على رابط خادم Laravel الحقيقي لتفعيل اللعب والمزامنة متعددة اللاعبين بصورة مركزية.

## تنبيه مهم

نسخة GitHub Pages ثابتة ولا تشغّل Laravel. لذلك توفر مسارات محلية للمعاينة والتجربة، بينما الغرف الحقيقية المشتركة، المحافظ المركزية، النوادي المتزامنة، الصلاحيات والإدارة الرسمية تتطلب نشر `backend-laravel` وربطه بالتطبيق.
